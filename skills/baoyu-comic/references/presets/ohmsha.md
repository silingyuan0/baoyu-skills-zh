# ohmsha

Ohmsha预设 - Educational manga with visual metaphors

## 基础配置

| 维度 | 值 |
|-----------|-------|
| 艺术风格 | manga |
| 基调 | neutral |
| 布局 | webtoon（默认）|

等同于：`--art manga --tone neutral`

## 独特规则

此预设包含超出 art+tone 组合的特殊规则。使用 `--style ohmsha` 时，必须应用以下所有规则。

### 视觉隐喻要求（关键）

每个技术概念都必须可视化为隐喻：

| 概念类型 | 可视化方法 |
|-------------|----------------------|
| 算法 | 展示过程的道具/机器 |
| 数据结构 | 角色可以进入/探索的物理空间 |
| 数学公式 | 环境中可见的变换 |
| 抽象过程 | 可触达的粒子/物体流动 |

**错误方法**：角色指着黑板解释
**正确方法**：角色使用"概念可视化道具"，进入隐喻空间

### 视觉隐喻示例

| 概念 | 错误（对话头像）| 正确（视觉隐喻）|
|---------|---------------------|------------------------|
| Attention mechanism | 角色指着黑板上的公式 | "注意力手电筒"道具在暗室中照亮关键词 |
| Gradient descent | "算法最小化损失" | 角色骑着球滚下山谷 |
| Neural network | 带箭头的图表 | 传递信息的发光生物活网络 |
| Overfitting | "模型记住了数据" | 角色穿着只适合特定姿势的衣服 |

### 角色设定（必需）

**默认：使用多啦A梦角色**，除非用户明确指定 `--characters` 或在 EXTEND.md 中有角色预设。

| 角色 | 默认角色 | 视觉 | 特征 |
|------|-------------------|--------|--------|
| 学习者（角色 A）| 大雄（Nobita）| 男孩，10岁，圆眼镜，黑发，黄衬衫，深蓝短裤 | 困惑，问基本但关键的问题，代表读者 |
| 导师（角色 B）| 哆啦A梦（Doraemon）| 蓝色机器猫，白色肚子，4D口袋，红鼻子，金色铃铛 | 知识渊博，耐心，用道具作为技术隐喻 |
| 挑战（角色 C）| 胖虎（Gian）| 矮胖男孩，小眼睛，橙衬衫 | 代表误解，或数据中的"噪声" |
| 支持（角色 D）| 静香（Shizuka）| 可爱女孩，黑色短发，粉红色连衣裙 | 问澄清问题，提供替代视角 |

**重要**：这些多啦A梦角色是 ohmsha 预设的默认角色。除非用户要求，否则使用这些确切的角色生成角色定义。

要使用自定义角色：`--characters "Student:小明,Mentor:教授"` 或在 EXTEND.md 中定义。

### 页面标题约定

每个页面必须有叙事标题（不是章节标题）：

**错误**："Chapter 1: Introduction to Transformers"
**正确**："The Day Nobita Couldn't Understand Anyone"

### 道具揭示模式

引入概念时：

1. 学生用视觉指示器表达困惑（？，螺旋眼）
2. 导师戏剧性地拿出带闪光效果的道具
3. 宣布道具名称，用粗体解释
4. 演示开始 - 学生进入隐喻空间

### 结尾要求

最后一页必须包括：

1. 学生展示理解（应用概念）
2. 回呼开场问题（现在已解决）
3. 导师满意的表情
4. 可选：暗示下一个主题

### 禁止对话头像规则

**关键**：角色必须做事情，而不仅仅是解释。

每个面板应显示：
- 执行的动作
- 演示隐喻
- 角色与概念空间的互动
- 不是：两个角色面对面交谈

### 特殊视觉元素

| 元素 | 用途 |
|---------|-------|
| 道具揭示 | 带闪光效果的戏剧性揭幕 |
| 概念空间 | 圆形边框，发光边缘为"想象模式" |
| 信息显示 | 全息 UI 风格显示技术细节 |
| 顿悟时刻 | 辐射线，闪光效果 |
| 困惑 | 螺旋眼，头上浮动的问号 |

## 质量标记

- ✓ 每个概念都是视觉隐喻
- ✓ 角色在做事，而不仅仅是说话
- ✓ 清晰的学生/导师动态
- ✓ 道具和道具驱动解释
- ✓ 表达性漫画风格情感
- ✓ 通过视觉设计而非文本墙显示信息密度
- ✓ 叙事页面标题
